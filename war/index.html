EmoTe API

About

The EmoTe API is a web service which calculates the various types and levels of emotions in a given passage of text. It exposes the synesketch.emotion package of the Synesketch Java API.

The API evaluates text and determines its general valence (ie., is it positive or negative?) and the strength (weight) of the general valence. The text will also have a series of emotions associated with it, each with their own weights.

Weights are on a 0-1 scale. A weight closer to 0 means that the emotion is very weak, whereas a value near 1 indicates that the emotion is very strong. Weights are assigned to the general valence of the text, as well as to each individual emotion present in the text.

The possible emotions are:

  • Happiness
  • Sadness
  • Fear
  • Anger
  • Disgust
  • Surprise

Usage

To use this API, please fork it on github and create a new GAE instance running this code. I would love to create one service to host everybody's requests, but I really don't feel like paying for that, or setting up a donation system to keep this running.

If you encounter any bugs, or would like any changes, please submit them at github.

Examples

Using POST

To use the API from a server-side technology, simply make a post request to the /api URI of your app engine instance (ie., http://emoteapi.appspot.com/api or http://localhost:8888/api) and include a parameter named "text" containing the text you'd like to evaluate. The server will return a JSON-encoded response.

JSONP (experimental)

JSONP support is experimental and takes two steps. First, you must make a request to the service to store the text you'd like to be evaluated in a cookie (note that if this is done through a GET request, there may be limitations as to how long your text string can be.) This request is sent to /api/store with a query parameter text containing the text. After the cookie has been set, create a script tag pointing to /api/jsonp with an optional callback parameter.
